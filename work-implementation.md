# মেডিসিন API প্রজেক্ট - কাজের সম্পূর্ণ বিবরণ (Work Implementation)

এই প্রজেক্টে শুরু থেকে এ পর্যন্ত কী কী কাজ করা হয়েছে এবং কীভাবে করা হয়েছে তার একটি পূর্ণাঙ্গ বিবরণ নিচে দেওয়া হলো, যাতে ভবিষ্যতে যে কেউ প্রজেক্টটি খুব সহজেই বুঝতে পারে।

---

## ১. ডাটাবেস ডিজাইন এবং মাইগ্রেশন (Database & Migrations)
আপনার আগের `Herbal_Brand` এবং `Alternate_Brand_Herbal` টেবিলের ওপর ভিত্তি করে আমরা দুটি নতুন এবং মডার্ন টেবিল তৈরি করেছি:
- **`medicines`**: মূল প্রোডাক্টগুলো রাখার জন্য।
- **`medicine_alternatives`**: অল্টারনেটিভ বা সাব-প্রোডাক্টগুলো রাখার জন্য।

এখানে আমরা ডাটাবেস কলামগুলোকে API এর জন্য সহজবোধ্য করেছি (যেমন: `tablet_name` কে `title`, `manufacturer_company` কে `brand` বানিয়েছি)। প্রাইসগুলোকে ডাটাবেসে `DECIMAL` (যেমন: 15.00) হিসেবে সেভ করার ব্যবস্থা করেছি যাতে পরবর্তীতে যেকোনো ক্যালকুলেশন সহজে করা যায়।

## ২. API তৈরি করা (CRUD Operations)
আমরা সম্পূর্ণ নতুন একটি API সিস্টেম তৈরি করেছি:
- **Models**: `Medicine` এবং `MedicineAlternative` মডেল তৈরি করা হয়েছে এবং তাদের মধ্যে রিলেশন (One to Many) সেট করা হয়েছে।
- **Service (`MedicineService`)**: সব লজিক এবং স্লাগ (slug) জেনারেট করার কাজ এখানে করা হয়েছে।
- **Controller (`MedicineController`)**: ডাটা রিসিভ এবং রিটার্ন করার জন্য `index`, `show`, `store`, `update`, `destroy` মেথডগুলো তৈরি করা হয়েছে।
- **Resource (`MedicineResource`)**: ডাটা ফ্রন্টএন্ডে পাঠানোর আগে সুন্দরভাবে সাজানোর কাজ করা হয়েছে। যেমন প্রাইসের সাথে `"৳ "` অ্যাড করে পাঠানো এবং ডিফল্ট থাম্বনেইল (thumbnail) সেট করা।
- **Pagination**: রেসপন্সের সাথে টোটাল পেইজ, বর্তমান পেইজ, এবং নেভিগেশন লিংকগুলো (Next, Prev) যুক্ত করা হয়েছে, যাতে ফ্রন্টএন্ড সহজে পেজিনেশন করতে পারে।

## ৩. ডাটা মাইগ্রেশন (Background Job & Scheduler)
পুরনো টেবিল থেকে হাজার হাজার ডাটা কোনো বাধা বা টাইমআউট ছাড়াই নতুন টেবিলে আনার জন্য আমরা ব্যাকগ্রাউন্ড প্রসেসিং তৈরি করেছি:
- **Job (`MigrateHerbalDataJob`)**: এটি পুরনো টেবিল থেকে १०० টি করে ডাটা একসাথে রিসিভ করে, প্রাইস থেকে `"৳ "` বা `"N/A"` রিমুভ করে ক্লিন করে এবং নতুন টেবিলে সেভ করে।
- **Command (`TriggerHerbalMigration`)**: `php artisan migrate:herbal-data` কমান্ডটির মাধ্যমে এই প্রসেসটি চালু করা যায়।
- **Scheduler**: প্রতিদিন রাত ১২টায় যেন এটি স্বয়ংক্রিয়ভাবে চলতে পারে, সেজন্য `routes/console.php` এ একে শিডিউল করে দেওয়া হয়েছে।

## ৪. API সিকিউরিটি (Static Token Authentication)
বাইরের কেউ যেন আপনার API তে বিনা অনুমতিতে হিট করতে না পারে, সেজন্য আমরা একটি স্ট্যাটিক টোকেন অথেন্টিকেশন তৈরি করেছি:
- **Config & .env**: `.env` ফাইলে `API_TOKEN_ENABLED=true` এবং `API_STATIC_TOKEN=your-token` সেট করা হয়েছে। এটি না থাকলে `config/api_settings.php` থেকে ডাটা নিবে।
- **Middleware (`VerifyStaticToken`)**: এই মিডলওয়্যারটি প্রতিটি API কলের হেডার চেক করে। হেডারে সঠিক টোকেন (`Authorization: Bearer your-token`) না থাকলে সে `401 Unauthorized` রিটার্ন করে।

## ৫. API ডকুমেন্টেশন (Scramble & Swagger UI)
সবশেষে, আপনার মোবাইল অ্যাপ বা ফ্রন্টএন্ড ডেভেলপারদের সুবিধার জন্য আমরা **Scramble (`dedoc/scramble`)** ইন্সটল করেছি।
- **কীভাবে কাজ করে?** Scramble স্বয়ংক্রিয়ভাবে আপনার কোড, রুলস এবং রিসোর্স পড়ে সুন্দর একটি Swagger UI তৈরি করে ফেলে। এর জন্য কোডের ভেতর বাড়তি কোনো কমেন্ট লিখতে হয় না!
- **UI কাস্টমাইজেশন:** `php artisan vendor:publish` কমান্ড ব্যবহার করে আমরা `config/scramble.php` ফাইলটি বের করেছি। সেখানে আমরা UI এর নাম "Medicine API Documentation" সেট করে দিয়েছি।
- **Local Network Access:** আপনি যেন আপনার মোবাইল বা অন্য পিসি (যেমন: `192.168.x.x`) থেকে এই ডকুমেন্টেশন দেখতে পান, সেজন্য `AppServiceProvider.php` ফাইলে আমরা একটি `Gate` তৈরি করে পারমিশন উম্মুক্ত করে দিয়েছি।
